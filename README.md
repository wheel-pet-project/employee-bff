# BFF (Backend for Frontend) - Employee Portal

BFF сервис для портала сотрудников системы каршеринга Wheel

## Описание

BFF (Backend for Frontend) сервис предоставляет единый REST API интерфейс для фронтенд приложений, агрегируя данные из различных микросервисов системы каршеринга. Сервис построен на основе API Gateway паттерна и обеспечивает оптимальный интерфейс для клиентских приложений.

## Архитектура

Проект следует принципам API Gateway архитектуры и состоит из следующих компонентов:

- **Gateway** - основной веб-приложение с REST API контроллерами
- **Clients** - библиотека gRPC клиентов для взаимодействия с микросервисами
- **Adapters** - адаптеры для преобразования между REST и gRPC контрактами
- **Middlewares** - промежуточное ПО для обработки запросов

## Основные функции

### 🔐 Аутентификация и авторизация
- **Аутентификация** - вход в систему по email и паролю
- **Подтверждение email** - активация аккаунта по токену подтверждения
- **Обновление токенов** - обновление access и refresh токенов
- **Восстановление пароля** - отправка и обработка токенов сброса пароля

### 🚗 Управление автопарком
- **Добавление транспортных средств** - регистрация новых автомобилей в системе
- **Просмотр автопарка** - получение списка всех транспортных средств
- **Детальная информация** - просмотр подробной информации о конкретном автомобиле
- **Поиск по геолокации** - поиск автомобилей в заданной области
- **Управление статусами** - изменение статуса транспортных средств

### 📋 Управление арендой
- **Создание аренды** - оформление новых договоров аренды
- **Просмотр аренд** - получение информации о текущих и завершенных арендах
- **Управление статусами** - изменение статуса аренды

### 📄 Документооборот
- **Управление документами** - загрузка и просмотр документов транспортных средств
- **Водительские удостоверения** - управление информацией о водительских правах
- **Проверки** - проведение различных проверок и валидаций

## Интегрированные микросервисы

BFF взаимодействует со следующими микросервисами через gRPC:

- **Identity** - аутентификация и управление пользователями
- **VehicleFleet** - управление автопарком и транспортными средствами
- **Rent** - управление арендой и договорами
- **VehicleDocuments** - документооборот транспортных средств
- **DrivingLicense** - управление водительскими удостоверениями
- **Check** - проведение проверок и валидаций

## API Endpoints

Сервис предоставляет следующие REST API endpoints:

### Аутентификация (`/api/identity`)
- `POST /authenticate` - аутентификация пользователя
- `POST /create-account` - создание нового аккаунта
- `POST /confirm-account-email` - подтверждение email адреса
- `POST /refresh-access-token` - обновление токенов доступа
- `POST /recover-password` - запрос на сброс пароля
- `POST /update-password` - обновление пароля

### Управление автопарком (`/api/vehicle`)
- `POST /add-vehicle` - добавление нового транспортного средства
- `GET /all-vehicles` - получение списка всех транспортных средств
- `GET /vehicle/{id}` - получение информации о конкретном автомобиле
- `GET /vehicle-details/{id}` - получение детальной информации
- `GET /vehicles-in-square` - поиск автомобилей в заданной области
- `DELETE /vehicle/{id}` - удаление транспортного средства

### Управление арендой (`/api/rent`)
- `POST /create-rent` - создание новой аренды
- `GET /rents` - получение списка аренд
- `GET /rent/{id}` - получение информации об аренде

### Документооборот (`/api/vehicle-documents`)
- `POST /upload-document` - загрузка документа
- `GET /documents/{vehicleId}` - получение документов транспортного средства
- `DELETE /document/{id}` - удаление документа

### Водительские удостоверения (`/api/driving-license`)
- `POST /create-license` - создание записи о водительских правах
- `GET /license/{id}` - получение информации о водительских правах
- `PUT /license/{id}` - обновление информации о водительских правах

### Проверки (`/api/check`)
- `POST /perform-check` - проведение проверки
- `GET /check-results` - получение результатов проверок

## Разработка

### Предварительные требования
- .NET 8 SDK
- Docker и Docker Compose
- IDE (Visual Studio, Rider, VS Code)

### Запуск локальной среды разработки

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd Carsharing.EmployeeBff
```

2. Запустите зависимые сервисы:
```bash
docker-compose -f up-local-environment-compose.yml up
```

3. Запустите BFF сервис:
```bash
cd Gateway
dotnet run
```

4. Откройте Swagger UI:
```
http://localhost:5000/swagger
```
